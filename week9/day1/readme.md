## 1. Day1

[binary search/DP]

- 예산1
  - 이진탐색 범위 갱신 시 left, right 기준이 아니라 mid를 기준으로!!
    - if(sum > K) right = mid - 1; //감소 시 상한선을 mid - 1
      else left = mid + 1;         //증가 시 하한선을 mid + 1
  - 단순히 로직을 구성하는 것이 아니라, 정답을 무엇을 어떻게 출력할 것인지도 고민
    - if(sum > K) right = mid - 1;
      else {
      answer = mid;
      left = mid + 1;
      }
    - 즉 예산이 k조건을 만족하였을때만 정답 후보군으로 저장해야한다.

- 예산2
  - 최소값, 최대값을 설정하였다. 그러나, 최대값보다 예산하한선이 더 클 경우
  - 기준을 지정하고 이분탐색을 진행하되, 그 합이 예산보다 같거나 작아야 한다.
    - 최대값보다 예산하한선이 더 클 경우, 이분탐색의 최대값이 예산하한선에 미치지 못하여 실제 지원가능 횟수보다 적어질 수 있음.
  - 이 경우 이분탐색의 기준을 어떻게 맞춰야 하는가?
    - 모든 예산을 더해서 그 합산이 하한선 이하일 경우 -> = N
    - 그렇지 못할 경우 -> 이분탐색
  - 이 문제는 이분탐색으로는 한계가 존재하여, DP로 해결해야 한다.
  - 정렬 후 (Arrays.sort(d)), 예산 선에서 작은것 모두 픽하기.

[접근방향]

- 하한선 지원가능? 지원아예불가? -> 이분탐색 : DP
- 실전에서 이러한 접근 방향이 떠오르지 않는다면, 기본 전략은 이분탐색.
  - 이 방향이 안되면 그때 DP
