[누적합(DP)]

- 구간합
  - 슬라이딩 윈도우는 불가능
  - 슬라이딩 윈도우라면, 각 합계구간이 연속되어 각 단계에서 누적활용 및 반영이 가능해야 한다.
  - DP와 약간 비슷한 접근, i번째 요소까지의 누적합을 "관리하는 것"이 핵심.
    - prefix (사실상 dp) = i"번째" 요소까지의 누적합 
    - from ~ to -> dp[to] - dp[from - 1]
  - 출력 누적이면 StringBuilder / 조합일 경우 별도의 arr까지..사용 의심

- 점프와 순간이동
  - 기본적인 접근 방법은 dp.
  - 선형크기 자체가 매우 크므로 선형 크기 자체를 줄일 수 있는 방안을 찾아야 한다.
    - 5 -> 4 -> 2
    - 10000 -> 5000 -> 2500 -> 1250 -> 625 
    - 10001 -> 1000 + 1
    - 10002 -> 5001 -> 5000 + 1
  
[DP 접근방향]

- 원소 요소 개수가 적당하다 -> 점화식 자체를 역방향 접근
  - dp[i] = from.Max or Min(가능한 경우의 수)
- 원소 요소 개수가 매우 많다 -> "해를 도출하는 방법"자체를 역방향 접근
  - 즉, 순방향으로 tabul을 채워가는 것이 아닌, tabul의 "연산 크기 자체"를 줄여나가는 것.

```java
while(N > 0)   //시작지점이 0, N = 0이 되었다면 종료, 연산 크기 자체는 만족할때만 진행하므로, 반씩 줄여나갈 수 있다.
if(N % 2 == 0) -> N = N % 2  //크기 자체를 줄여나간다.
else -> N = N - 1 / answer ++ //크기 자체를 줄여나간다.
```
