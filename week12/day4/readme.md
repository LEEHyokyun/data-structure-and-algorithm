- 스티커
  - 요소 10만개 BFS 가능? 불가 (무조건 1번만 순회해야 메모리 허용 범위 내 연산 가능)
  - 1개 선택시 다음 선택지 정해져있다 -> 점화식 DP
    - 열을 기준으로 순회
      - dp[0][i] -> i번째 열에서 0번째 행의 스티커를 뜯을때 - i-1번째 열에서 1,2 중 최대값 + 현재 스티커
        - 초기값 dp[0][0] = graph[0][0]
      - dp[1][i] -> i번째 열에서 1번째 행의 스티커를 뜯을때 = i-1번째 열에서 0,2 중 최대값 + 현재 스티커
        - 초기값 dp[0][1] = graph[0][1]
      - dp[2][i] -> i번째 열에서 아무것도 뜯지 않을때 = i-1번째 열에서의 최대값 그대로 (Math.max(dp[2][i-1], Math.max(dp[0][i-1], dp[1][i-1])))
        - 초기값 dp[0][2] = 아무것도 안뜯으므로 합산하지 않고 0이다.
