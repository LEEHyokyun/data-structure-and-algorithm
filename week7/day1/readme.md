- 회의를 몇개 선택할 수 있는가

  - 그 이후에 선택하는 회의시간이 이전 회의의 종료시간보다 더 큰 회의 선택.

  - 단순 가벼운 사람 기준 정렬한 후 가벼운 순서대로 선택하면 반례있어 적용 불가 
  - 정렬 후, 가벼운사람 + 무거운사람이 limit이하면 가벼운사람 + 무거운사람, 초과면 무거운사람
    - 무거운 사람을 반드시 태운다는 점이 핵심...start/end 조절을 통해 태우는 방법.
    - limit초과하면 가벼운 사람을 오히려 뺀다.

[Arrays.sort 정렬 구성하기]

- 요약

| 리턴값      | 의미     | 정렬 결과        |
| -------- | ------ | ------------ |
| 음수 (< 0) | a < b  | **a가 앞에 온다** |
| 0        | a == b | 순서 유지        |
| 양수 (> 0) | a > b  | **b가 앞에 온다** |

- 오름차순/내림차순
  - a-b -> 음수면 a-b / 양수면 b-a (즉 작은순서 = 오름차순)
    - | a | b | a - b | 결과 | 정렬    |
      | - | - | ----- | -- |-------|
      | 3 | 5 | -2    | 음수 | 3 → 5 |
      | 5 | 3 | 2     | 양수 | 3 → 5 |
      | 4 | 4 | 0     | 같음 | 그대로   |

  - b-a -> 음수면 b-a / 양수면 a-b (즉 큰 순서 = 내림차순)
    - | a | b | b - a | 결과 | 정렬    |
      | - | - | ----- | -- | ----- |
      | 3 | 5 | 2     | 양수 | 5 → 3 |
      | 5 | 3 | -2    | 음수 | 5 → 3 |
