## 1. Day5

[DP(Tabulation)]
- Implements : 배운점
  - 문제 정독 필수
    - 1234567 나머지 연산
  - 문제유형을 파악한다.
    - 조합의 방법의 수
    - 모든 경로를 탐색하는 것이 아닌 방법의 수를 도출
      - +1 일 경우 -> dp[n-1]
      - +2 일 경우 -> dp[n-2]
        - 이에 대한 경우의 수를 모두 합산 
    - 짝수일 경우에만 방법의 수를 도출
      - 이전 짝수의 방법을 3가지로 조합
- Improvements : 막혔던 지점 / 고민한 포인트
  - 기본적으로 문제에서 특정 수로 나눈다면, 반환 형태를 int가 아닌 long 형태로 할 것.
    - 웬만하면 tabulation
    - 문제가 모든 경우의 수를 탐색하는 것이 아닌 "조합의 수"
      - +1일때와 +2일때...DP를 적용
