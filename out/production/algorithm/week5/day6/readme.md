## 1. Day6

[back tracking(DFS)/문자열 연산 종합]
- Implements : 배운점(문제)
  - 큰수에 곱하기를 적용하고, 작은수에 더하기나 빼기를 적용하는 그리디는 반례가 있어서 불가
    - 애초에 문제 조건이 "연산 우선순위를 무시하고 앞에서부터 차례대로 연산"한다는 조건이 있다.
    - 예를 들어 345의 경우 3+4*5가 아닌 (3+4)*5가 된다.
  - 따라서 모든 경우의 수를 dfs로 탐색하되, 특정 조건 만족시 결과를 반환하는 백트래킹 방식을 적용한다.
  - 모든 수에 대해 연산을 적용하기 위해선 dfs로 끝내지 말고, 반드시 연산횟수 복구를 해주어야 한다.
    - i=0에서 i=1로 넘어가는 순회 시(즉 다음순회로 넘어갈때 = i=0일때의 dfs 완료) 연산자가 복구가 되어있어야 다른 숫자에 적용 가능
      - ex 3 4 5
      - i = 0일때 순차적용
      - i = 1일때 최초 적용 후, 그 다음 dfs는 idx/next값 상관없이 for 순회를 통해 op적용
      - 다음 순회를 위해 반드시 복구해주어야 한다(즉 큰 순회가 넘어가더라도, 이에 상관없이 dfs는 연산을 모두 순회함).
  - 문자열 연산 종합
    - toCharArray(), String -> length(), StringBuilder + sb.append() + sb.toString()
      - balanced -> 수
      - proper -> 수 + 적절성 여부 판단
        - Stack을 통해 복잡한 구현보다, 단순 bal++/bal--를 통해 판단 가능하며
        - 끝까지 순회 간 bal == 0 일 경우만 OK, 중간에 0이 되거나 끝까지 순회 시 bal > 0일 경우는 모두 NG.
- Improvements : 작동원리/개선포인트(본질)
  - 모든 연산 결과의 비교 후 최대값 및 최소값 추출이 "필요하다면"
    - 백트래킹
  - 문자열 연산 종합에 대한 내용
