## 1. Day5

[슬라이딩 윈도우]
- Implements : 배운점(문제)
  - 슬라이딩 윈도우 = 고정된 크기의 합산
  - 투포인터 = 고정된 크기를 합산에 따라 조절해나간다.
    - 슬라이딩을 적용한 이후에는 크거나 작아진다.
      - sum > target -> left를 뺀다. + left++
      - sum < target -> right를 더한다. + right++
        - 이후 단계에서 sum > target or sum < target은 자동적용, 이에 따라 슬라이딩 자동 적용.
          - sum = target -> count ++
          - 다음 단계를 위해 left를 뺀다.
          - left = left + 1.
  - 슬라이딩 윈도우의 크기가 일정치 않아도 된다는 점이 중요하다.
    - 다음 단계에서 자동 적용.
  - 두번째 케이스도 "연속된 수의 합" = 슬라이딩 윈도우 적용.
    - 크면 왼쪽 제거, 작으면 오른쪽 추가하는 방식으로 첫번째 케이스 그대로 적용.
- Improvements : 작동원리/개선포인트(본질)
  - 특정 두 수의 합 = 투포인터
    - 정렬을 한다.
      - 수 합산 시 두개로 고정되어있다.
      - 합산에 따라 범위를 조정해나간다(tar > sum -> left + 1, tar < sum -> right - 1)
  - 수열의 연속된 크기의 부분합 = 슬라이딩 윈도우
      - 정렬을 하지 않고 주어진 배열 그대로 사용한다.
      - 연속된 크기의 부분합인 슬라이딩 윈도우
      - 이와 동시에 고정된 크기를 합산에 따라 조정해나가는 투포인터
        - 합산에 따라 범위를 조정해나간다(tar > sum -> left + 1, tar < sum -> right - 1)

- Additional : 추가적인 유의점들

- 기존 고정된 크기를 늘려나가는 방법 -> O(N^2)\
  - brute force는 절대 아니다.
- 이 문제는, 모든 수는 양수 / sum은 윈도우 크기 증가 시 증가 / 같은 구간을 다시 볼 필요 없음
  - 다음 단게에서 이전의 단계에 따라 슬라이딩을 그대로 적용한다는 것
  - 슬라이딩 크기가 일정치 않아도 된다는 점
  - O(N) 시간복잡도로 해결 가능
